# VA Queue â€” Blog Review & Social Publishing System

## Overview

The VA Queue is a human-in-the-loop review layer that sits between automated blog post generation and public publishing. Every post generated by the daily or weekly pipelines starts with `workflowStatus: 'media_pending'` and must be approved here before it goes live.

The queue also handles the full video pipeline: generating a video script, creating a base talking-head video via HeyGen, uploading a final edited video, and publishing it to all of Shana's connected social channels via Blotato.

**URL:** `/admin/va-queue/`
**Auth:** Same HMAC session cookie as the rest of admin (`sg_assistant_session`)

## Video Pipeline

### Step 1 â€” Generate Script

**Button:** "Generate Script" in the Video Script card
**API:** `POST /api/content/generate-script`
**Model:** `claude-haiku-4-5-20251001`
**Voice:** Shana Gates, Coachella Valley REALTORÂ® â€” warm, knowledgeable, conversational. 45â€“90 seconds when read aloud. Covers key takeaways, what it means for the CV market, and a call to action.
**Input:** post title, excerpt, category
**Output:** plain text script in the textarea (editable before use)

### Step 2 (optional) â€” Generate Base Video with HeyGen

**Button:** "Generate with HeyGen" in the Video card
**API:** `POST /api/content/generate-heygen-video` â†’ polls `GET /api/content/heygen-status`
**How it works:**
1. Sends script to HeyGen `/v2/video/generate`
2. Returns `videoId`; editor polls `/api/content/heygen-status?videoId=` every 15 seconds
3. On completion, HeyGen video is streamed into Vercel Blob for a permanent URL
4. Editor shows a download link â€” VA downloads, edits (adds b-roll, graphics, etc.), then uploads the final cut in Step 2

**Video format:** 720Ã—1280 vertical (9:16) â€” optimised for Shorts, TikTok, Reels
**Background:** `#1a1a1a` dark neutral
**Poll timeout:** 10 minutes

**HeyGen API notes:**
- Auth header: `X-Api-Key: {HEYGEN_API_KEY}` (not `Authorization: Bearer`)
- API key type: **Agent** (not Developer) â€” Developer keys have restricted access
- Credits: HeyGen uses separate **API credits** from studio credits. API credits must be purchased as an add-on even on paid plans (Pro+)
- `HEYGEN_AVATAR_LOOK_ID` = the avatar look ID (specific outfit/appearance), found on the avatar detail page in HeyGen
- `HEYGEN_VOICE_ID` = voice ID for speech synthesis, found in the HeyGen voice library

### Step 3 â€” Upload Final Video

**Input:** `<input type="file">` â€” MP4, MOV, or WebM up to 500 MB
**API:** `GET /api/content/upload-token` â†’ browser XHR PUT directly to Vercel Blob
**How it works:**
1. Server generates a client upload token via `generateClientTokenFromReadWriteToken()` from `@vercel/blob/client`
2. Browser does a raw XHR `PUT` to Vercel Blob CDN with `Authorization: Bearer {token}` and `x-content-type` headers
3. XHR `onprogress` drives the upload progress bar â€” no server relay needed
4. On success, `pendingVideoUrl` is set to the permanent Vercel Blob URL

### Step 4 â€” Mark as Ready

**Button:** "Mark as Ready"
**API:** `POST /api/blog/queue-mark-ready`
**Saves:** title, socialCopy, heroImageBase64 (thumbnail), videoScript, videoUrl, videoThumbnailUrl
**Effect:** Sets `workflowStatus: 'media_ready'` in both the full post key and queue summary in Redis

### Step 5 â€” Publish

**Button:** "Publish" (only enabled when status is `media_ready`)
**What it does:**
1. Saves all fields via `queue-mark-ready`
2. Calls `POST /api/blog/queue-publish` â†’ moves post to `published`, removes from queue, adds to public blog index
3. If a video is attached: calls `POST /api/content/publish-video` â†’ publishes to all 7 Blotato platforms in parallel
4. Platform status grid shows live polling results per platform

## Social Publishing via Blotato

**API:** `POST /api/content/publish-video`
**Blotato endpoint:** `https://backend.blotato.com/v2/posts`
**Auth header:** `blotato-api-key: {BLOTATO_API_KEY}`

Publishes in parallel to 7 platforms:

| Platform | Blotato targetType | Account ID env var |
|---|---|---|
| Facebook Reel | `FACEBOOK_REEL` | `BLOTATO_FACEBOOK_ACCOUNT_ID` + `BLOTATO_FACEBOOK_PAGE_ID` |
| YouTube | `YOUTUBE` | `BLOTATO_YOUTUBE_ACCOUNT_ID` |
| TikTok | `TIKTOK` | `BLOTATO_TIKTOK_ACCOUNT_ID` |
| LinkedIn | `LINKEDIN` | `BLOTATO_LINKEDIN_ACCOUNT_ID` |
| X / Twitter | `TWITTER` | `BLOTATO_X_ACCOUNT_ID` |
| Threads | `THREADS` | `BLOTATO_THREADS_ACCOUNT_ID` |
| Instagram Reel | `INSTAGRAM_REEL` | `BLOTATO_INSTAGRAM_ACCOUNT_ID` |

**Caption building:**
- If post has `socialCopy`, it's used as the base for all platforms
- Otherwise `generatePlatformCaptions()` in `lib/publish-service.ts` generates platform-specific copy via Claude Haiku
- Each platform caption is built by a dedicated builder function that enforces character limits and appends the blog post URL and relevant hashtags

**Coachella Valley hashtags (appended per platform):**
```
#CoachellaValley #PalmSprings #PalmDesert #RanchoMirage #LaQuinta #IndianWells #ShanaGates #CraftAndBauer
```

**Platform character limits:**
- Facebook: 2000
- LinkedIn: 3000
- X/Twitter: 257 (leaves room for URL)
- Instagram: 2200
- Threads: 500

**Status polling:**
Each platform returns a `postSubmissionId`. The editor polls `GET /api/content/blotato-status?postSubmissionId=` every 10 seconds for up to 5 minutes to track final publish status.

## Redis Data Model

Posts are stored in Upstash Redis (not Sanity) for the queue:

| Key | Value | Notes |
|---|---|---|
| `blog_post:{slug}` | Full `BlogPostFull` JSON | Every field including body, videoScript, videoUrl |
| `blog_posts_queue` | Array of `BlogPostSummary` JSON | Posts in `media_pending` or `media_ready` state |
| `blog_posts_index` | Array of `BlogPostSummary` JSON | Published posts only â€” powers public blog listing |

**`BlogPostFull` fields (relevant to VA queue):**
```typescript
workflowStatus: 'media_pending' | 'media_ready' | 'published'
socialCopy?: string       // Facebook caption + base for all social platforms
heroImageUrl?: string     // Vercel Blob URL for thumbnail
videoScript?: string      // Script text
videoUrl?: string         // Vercel Blob URL for final video
videoThumbnailUrl?: string // Optional separate video thumbnail
title: string             // Editable in the VA queue editor
```

## Known Gotchas

- **HeyGen 401 "internal_error"** â€” key is wrong or revoked. Regenerate from HeyGen Settings â†’ API.
- **HeyGen "Insufficient credit"** â€” API credits exhausted. Purchase more from Billing â†’ API Credits. Studio credits are a separate pool.
- **Video upload 401** â€” Vercel Blob token not set or expired. Check `BLOB_READ_WRITE_TOKEN`.
- **Blotato platform shows "Error"** â€” hover the status pill for the raw error. Usually means the account ID for that platform is wrong or the platform connection expired in Blotato.
- **`[object Object]` in error** â€” should be fixed; all HeyGen errors are now stringified via `JSON.stringify()` before display.
